What's New?

The following new features were added in 1.0.67:

  • SEA/LEA end users only able to write data within own EdOrg hierarchy - The SLI, through the API, allows access of student's data across EdOrgs in cases when a student is enrolled in more than a one EdOrg. Because the roles are not aware of the context, scenarios existed where data that does not belong to the user's EdOrg (hierarchy) can be edited. This feature disables any writing to data that is not in the user's EdOrg hierarchy.

Updates to the Ingestion Schema

Ingestion Schema Changes

We have modified the ingestion schema, in many cases using extensions and restrictions. The following section describes those changes.

Domain Entities

  • EducationalOrgReference - Removed the REF attribute, as ID/REF are no longer supported.

No changes to the API were implemented in 1.0.67

Resolved Issues

The following is a list of previously reported issues which have been resolved in the 1.0.67 release. Click defect title to see additional description:

DE2325: Ingestion does not report errors if a ctl file is unzipped incorrectly.

A zip file is ingested and unzipped incorrectly so that the ctl is no longer present. The ingestion job will complete, but it will not produce a job log file or a error file. This issue was resolved by this fix.

DE1876: Posting an entity twice results in API 500 error

Posting an entity twice resulted in an API 500 error. We had changed features of the natural keys calculations that caused this error. We have fixed this defect.

DE2339: Incorrect error resource ID stored

Incorrect resourceID of "studentAssessmentItem" in error messages. This should have been the corresponding AssessmentMetaData XML file from which it originated. As a result, no error files were created, and the job was erroneously reported as having all records processed successfully. With this fix this issue had been corrected.

DE2162: Sandbox Administrator (role) cannot see Admin tab in Portal

This issue has been resolved. Previously, as a workaround, the user could be explicitly assigned the Application Developer role to resolve the issue.

DE2400: Cannot associate a student to multiple cohorts

Fixed error in the determination of grace period when an end date is not specified for an entity.

Known Issues

The following are the defects currently used to track known issues in the 1.0.67 release. Click a link below for more information:

DE921: Duplicates are not reported

Error reports are not always clear regarding the records persisted. When there are duplicates within the XML file, or a record already exists, the ingestion engine will still update the record. However, this record count is not included in the Ingested records count.

DE1458: Handle more than one teacher of record

The ingestion service can currently accept more than one teacher for a single section. The dashboard does not currently support this behavior.

DE157: School and EdOrg only supports a single representation of parentEducationAgencyReference

Reference resolution is only supported as the default scenario that features no exceptional or error conditions. As a result, a user may ingest valid EdFi format with improper references that would fail to resolve.

DE325: Global uniqueness of ProgramIdentifier is assumed when multiple EdOrgs can use the same IDs

Ingestion process assumes that the Program ID is unique within a tenant, but this should be unique within the specified EdOrg instead. As a result, the reference could resolve incorrectly.

DE273: DisciplineDescriptor not currently supported in Interchange-StudentDiscipline

This descriptor is not currently supported. However, a reference to this descriptor is required for the DisciplineAction entity.

DE198: EducationOrganization references to Program not properly ingested

This reference hasn't been implemented yet. As a result, the API won't provide a link from the EdOrg to the Program.

DE1434: Dashboard Builder - Drag and drop on release doesn't always return the page to the right state

When a large dataset is available, the initial dashboard view may only display a subset of the data. The dashboard currently does not display a message explaining that only a subset of data is displayed.

DE1647: Dashboard may not display all data

When a large dataset is available, the initial dashboard view may only display a subset of the data. The dashboard currently does not display a message explaining that only a subset of data is displayed.

DE1457: Section Profile - Handle 403 error when making teacher call

Currently, making a teacher call for a section with two "teachers of record" returns a 403 error. This error can cause the sectionInfo panel to not display.

DE1573: Remove Attendance Rate

The current attendence record includes an attendance rate column that may not be accurate for some use cases. The column will be removed by default in a future release.

DE162: Course name (not course code) is used to uniquely identify and refer to Course entities

Course entities are uniquely identified by course name within an education organization, rather than by course code.

DE710: Section, school and session references in AttendanceEvent are ignored

The way AttendanceEvent entities are handled ignores explicit characterization of the attendance event - by school, section, session. As a result, AttendanceEvents for students attending multiple schools will not properly persist.

DE2160: Dashboard: Select filter after listing section removes all student data

When using SLC Dashboard and viewing a list of students, in some cases choosing a filter related to that student's "special needs" status will result in a "no students found" error.

DE2173: Truncation limit for ingestion errors and warnings misconfigured in SLI Properties

Each error file is truncated after the maximum number of warnings and the maximum number of errors have been reached. The maximum number of errors is 10000 per input file. The maximum number of warnings is 10000 per input file.

DE2314: Admin or API should handle invalid EdOrgs when saving an application in the Sandbox

When an Application Developer saves an application in the Sandbox and that application has an existing allowed_edorg that no longer exists, then you get a 400 error "edorg is outside your tenant".

DE2079: Update HATEOS links

In some cases the HATEOS links that the API returns may not work as expected. The contextual URI rewrites convert some API requests and some of these converted requests may not return what is expected.

DE2289: Randomness with custom configuration

In Dashboard, some users are seeing Middle School View in the "Views" dropdown even when not viewing middle school students.

DE2173: Truncation limit for ingestion errors and warnings misconfigured in SLI properties file

This reference hasn't been implemented yet. As a result, the API won't provide a link from the EdOrg to the Program.

DE2355: The Administrator Account Management Tool allows LEA admin creation at multiple levels of the LEA edorg hierarchy

The Administrator Account Management Tool allows creation of non-top LEA level users. In case when multiple levels of education organizations of type LEA are ingested in the system. Even though a creation of such user will be successful, multiple features of the system are not supported for LEA Admins associated with a sub-LEA (admin delegation and application authorization). LEA Administrator should be created only for the top level of LEA, in cases when multiple level of LEAs exist
