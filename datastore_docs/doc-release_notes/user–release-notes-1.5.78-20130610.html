inBloom technology 1.5.78 Release Notes

What's New?

Bulk Extract - Purge Events

Bulk Extract's Delta Deletes file reports purge events performed through the ingestion system. Purge events are reported differently than deletes events. For details, see documentation for the bulk extract API.

Bulk Extract - Error Reporting

Basic error reporting metrics are captured during the creation of the bulk extract data files. In the same directory as the data files, error.txt is created to report errors that occurred during bulk extraction.

The error.txt file indicates that errors occurred during the process and the latest successful full extract should be obtained. A list is included to provide more specific information about the type of errors that were encountered. The file lists the number of errors that occurred per entity type.

Example: 2 errors occurred for entity type student

Bulk Extract - Full (top-level) LEA Extract

The following entities are now supported in the full (top-level) LEA Extract: Cohort, StudentCohortAssocation, StaffCohortAssocation, Section, StudentSectionAssocation, TeacherSectionAssocation, GradebookEntry, StudentGradebookEntry, StudentCompetency, CourseTranscript, DisciplineAction, DisciplineIncident, StudentDisciplineIncidentAssociation, and GraduationPlan. For a full list of supported entities, see the API documentation.

Bulk Extract - Delta (top-level) LEA Extract

The following entities are now supported in the delta (top-level) LEA Extract: GradingPeriod, Session, Course, CourseOffering, GraduationPlan, DisciplineIncident, DisciplineAction, StudentDisciplineIncidentAssociation, StudentCompetency, and CourseTranscript. For a full list of supported entities, see the API documentation.

Bulk Extract - List API

The List (bulk/extract/list) API includes the link to the full SEA Extract.

Data Model Updates

This release includes the following updates to the data model:

  • Discrete language and identificationSystem fields have been added under the following entities/elements. The definitions for these new fields mirror those added to the ingestion schema:
    • For Student: languages and homeLanguages
    • For StudentAssessment: administrationLanguage

Ingestion Schema Updates

This release includes the following updates to the ingestion schema, listed by entity type:

  • SLC-LanguageItemComplexType - replaces LanguageItemType in Ed-Fi Core as follows:
    • Changes enumeration to a string with minimum length of 1 and maximum length of 75
    • Adds an optional attribute, IdentificationSystem, that is a string with a minimum length of 1 and a maximum length of 60
    • Applies these changes to the following entities:
      • For Student: Languages and HomeLanguages
      • For Student Assessment: AdministrationLanguage

API Updates

This release includes the following changes to the API:

Added

No end points have been added.

Deprecated

No end points have been deprecated

New and Updated Fields

  • program.programId max length has increased to 60 characters
  • student.languages.language (formerly student.languages)
  • student.languages.identificationSystem (added)
  • student.homeLanguages.language (formerly student.homeLanguages)
  • student.homeLanguages.identificationSystem (added)
  • studentAssessment.administrationLanguage.language (formerly studentAssessment.administrationLanguage)
  • studentAssessment.administrationLanguage.identificationSystem (added)

Resolved Issues

The following is a list of previously reported issues which have been resolved this release.

DE2826: Forced Deletion, Update or Insert of Ed Orgs may cause inappropriate access through the API

Forced Deletion, Update or Insert of Ed Orgs may cause inappropriate access through the API.

DE2835: When checking for expired status, today is considered expired

When determining access to data through an association (e.g. StaffEducationOrgAssignmentAssociation), in cases when the association's end date is 'today', the association is considered expired and the user loses access. It should be considered as current.

DE2867: Bulk Extract: Full Extract missing staffEdOrg references when staff in multiple LEAs

When Staff is in multiple EducationOrganizations (has multiple StaffEducationOrgAssignmentAssociations), the full (top-level) LEA extract includes just one StaffEducationOrgAssignmentAssociation.

Known Issues

The following are the open defects currently used to track known issues in this release.

DE198: EducationOrganization references to Program are not properly ingested.

This reference is not implemented yet. As a result, the API does not provide a link from the EdOrg to the Program.

DE273: A DisciplineDescriptor is not currently supported in Interchange-StudentDiscipline.

This descriptor is not currently supported. However, a reference to this descriptor is required for the DisciplineAction entity.

DE325: Global uniqueness of ProgramIdentifier is assumed when multiple EdOrgs can use the same IDs.

The ingestion process assumes that the Program ID is unique within a tenant, but this should be unique within the specified EdOrg instead. As a result, the reference could resolve incorrectly.

DE408: Aggregate Viewers cannot see themselves.

Users with the Aggregate Viewer role cannot see their own Teacher or Staff entity.

DE710: Section, school and session references in AttendanceEvent are ignored.

The way AttendanceEvent entities are handled ignores explicit characterization of the attendance event - by school, section, session. As a result, AttendanceEvents for students attending multiple schools do not properly persist.

DE840: inBloom data model does not account for Ed-Fi Core v. 1.0.03 version of Address with Country and AddressLine

The Address entity changed in EdFi 1.0.03 to have separate structures to represent addresses that specify "Country" with generic "AddressLine"s. Ingestion does not currently account for this change and drops country code and the international address lines.

DE921: Duplicates are not reported.

Error reports are not always clear regarding the records persisted. When there are duplicates within the XML file, or a record already exists, the ingestion engine still updates the record. However, this record count is not included in the ingested records count.

DE1434: Internet Explorer 8 users may experience issues when using the dashboard builder.

If you are using Internet Explorer 8 and the dashboard builder, then intermittently the release after a drag and drop does not return the page to the correct state.

DE1457: Making a teacher call for a section with two "teachers of record" returns a 403 error.

Making a teacher call for a section with two "teachers of record" returns a 403 error. This error can cause the sectionInfo panel to not display.

DE1458: The ingestion service is able to handle more than one teacher of record.

The ingestion service can accept more than one teacher for a single section. The dashboard does not currently support this behavior.

DE1573: Attendance Rate should be removed.

The current attendance record includes an attendance rate column that may not be accurate for some use cases. The column could be removed by default in a future release.

DE1647: Dashboard may not display all data.

When a large dataset is available, the initial dashboard view may only display a subset of the data. The dashboard currently does not display a message explaining this scenario.

DE1728: An error occurs if an OAuth Callback URL does not begin with "HTTPS".

If an application includes a callback URL that does not begin with "https://", then an error message is returned.

DE2079: HATEOS links don't always work as expected.

The HATEOS links that the API returns may not work as expected in all cases. The contextual URI rewrites convert some API requests. Some of these converted requests may not return what is expected.

DE2160: Selecting a filter after listing a section removes all student data.

When viewing a list of students using the inBloom Dashboard, choosing a filter related to that student's "special needs" status results in a "no students found" error.

DE2300: There is a 250 result limit on global entities contained in elasticsearch.

When querying the following entities, results are limited to a maximum of 250 returns:

  • /assessments
  • /competencyLevelDescriptor
  • /learningObjectives
  • /learningStandards
  • /studentCompetencyObjectives

DE2725: Updating an entity does not correctly update the search index.

When an existing entity in the system is updated, that entity is not updated in elasticsearch and becomes unsearchable.

DE2833: Deletes from Interchanges aside from InterchangeAssessmentMetadata does not work for ObjectiveAssessement and possibly others.

The Action entity type can be present in any interchange and should support deletion of any entity type. Currently, the Action entity with the type attribute set to DELETE will only support deletion of entities within the interchange in which the Action entity is present. 

DE2851: Newly created student entity is unsearchable.

After creating a student entity, it cannot be retrieved the from API endpoint.

DE2852: Updating a learningObjective entity does not correctly update search index.

When an existing entity in the system is updated, that entity is not updated in elastic search and becomes unsearchable.

DE2853: Updating a learningStandard entity does not correctly update search index.

When an existing entity in the system is updated, that entity is not updated in elastic search and becomes unsearchable.

DE2857: Updating a courseOffering entity does not correctly update search index.

When an existing entity in the system is updated, that entity is not updated in elastic search and becomes unsearchable.
