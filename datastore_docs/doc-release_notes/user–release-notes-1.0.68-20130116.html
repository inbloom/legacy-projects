What's New?

The following new features were added in 1.0.68:

  • Student search API - Our new student search API is available as of this release. It allows developers to build applications that allow users to query for specific students and is more flexible than the legacy search and filter functionality. Five additional entities are now available to query via the search capability.
    • /assessments
    • /competencyLevelDescriptor
    • /learningObjectives
    • /learningStandards
    • /studentCompetencyObjectives
  • Multiple API versions - Multiple API versions are now supported. See inBloom for Developers for complete information on the multiple versions of the API.
  • StaffEducationOrgAssignmentAssociation required for Teachers - A StaffEducationOrgAssignmentAssociation is required for every Teacher entity in the system. This additional requirement ensures that the data is considered valid.
  • Redesigned Sanbox Login - The Realm Chooser and Login pages in Sandbox are now one page. This allows the application developer to easily navigate between the administrative and impersonation mode of the Sandbox.
  • Unified Accounts - Now when a developer registers for an inBloom Sandbox account, they can use that account to also log in to an inBloom Production portal.

Resolved Issues

The following is a list of previously reported issues which have been resolved in the 1.0.68 release.

DE2235: The inBloom Dashboard does not display course information for those courses that have EdOrg references to LEAs

Course information for courses that have EdOrg references to LEAs could not be displayed. This is fixed.

DE2354: Minimizing the risks originating from Mongo defects

The system verifies that neither the field name nor the value contains any characters that Mongo interprets as a new line.

DE2346: Attendance calendars can show different months for linda.kim and jstevenson

There were scenarios where different attendance calendars displayed different time ranges during the same school year depending on the user. This is fixed.

DE2400: Cannot associate a student to multiple cohorts

Scenarios could occur where a student could not be associated to multiple cohorts. This is fixed.

Known Issues

The following are the defects currently used to track known issues in the 1.0.68 release.

DE921: Duplicates are not reported

Error reports are not always clear regarding the records persisted. When there are duplicates within the XML file, or a record already exists, the ingestion engine still updates the record. However, this record count is not included in the ingested records count.

DE1458: The ingestion service is able to handle more than one teacher of record

The ingestion service can accept more than one teacher for a single section. The dashboard does not currently support this behavior.

DE157: School and EdOrg only support a single representation of parentEducationAgencyReference

Reference resolution is only supported as the default scenario that features no exception or error conditions. As a result, a user may ingest valid EdFi format with improper references that fail to resolve.

DE325: Global uniqueness of ProgramIdentifier is assumed when multiple EdOrgs can use the same IDs

The ingestion process assumes that the Program ID is unique within a tenant, but this should be unique within the specified EdOrg instead. As a result, the reference could resolve incorrectly.

DE273: A DisciplineDescriptor is not currently supported in Interchange-StudentDiscipline

This descriptor is not currently supported. However, a reference to this descriptor is required for the DisciplineAction entity.

DE198: EducationOrganization references to Program are not properly ingested

This reference is not implemented yet. As a result, the API does not provide a link from the EdOrg to the Program.

DE1647: Dashboard may not display all data

When a large dataset is available, the initial dashboard view may only display a subset of the data. The dashboard currently does not display a message explaining that this scenario.

DE1457: Section Profile - Handle 403 error when making teacher call

Making a teacher call for a section with two "teachers of record" returns a 403 error. This error can cause the sectionInfo panel to not display.

DE1573: Remove Attendance Rate

The current attendance record includes an attendance rate column that may not be accurate for some use cases. The column could be removed by default in a future release.

DE162: Course name is used to uniquely identify and refer to Course entities

Course entities are uniquely identified by course name within an education organization, rather than by course code.

DE710: Section, school and session references in AttendanceEvent are ignored

The way AttendanceEvent entities are handled ignores explicit characterization of the attendance event - by school, section, session. As a result, AttendanceEvents for students attending multiple schools do not properly persist.

DE2160: Selecting a filter after listing a section removes all student data

When using the inBloom Dashboard and viewing a list of students, choosing a filter related to that student's "special needs" status results in a "no students found" error.

DE2173: Truncation limit for ingestion errors and warnings is misconfigured

Each error file is truncated after the maximum number of warnings and the maximum number of errors are reached. The maximum number of errors is 10000 per input file. The maximum number of warnings is 10000 per input file.

DE2314: Admin or API should handle invalid EdOrgs when saving an application in the Sandbox

When an application developer saves an application in the sandbox and that application has an existing allowed_edorg that no longer exists, then you get a 400 error "edorg is outside your tenant".

DE2079: Update HATEOS links

The HATEOS links that the API returns may not work as expected in all cases. The contextual URI rewrites convert some API requests. Some of these converted requests may not return what is expected.

DE2300: There is a 250 result limit on Global Entities contained in elasticsearch

When querying the following entities, results are limited a maximum of 250 returns: /assessments, /competencyLevelDescriptor, /learningObjectives, /learningStandards and /studentCompetencyObjectives.

DE408: Aggregate Viewers cannot see themselves

Users with the Aggregate Viewer role cannot see their own Teacher or Staff entity.

DE2333: Admin Delegation for Application Approval is not showing all applications

Delegating Application Authorization is not working correctly.

DE1728: An error occurs if OAuth Callback URL does not begin with "https"

If an application includes a callback URL that does not begin with "https://", then an error message is returned.

DE2398: Installed Application Authentication Support

It is unclear to the client how the authentication process works for 'installed' appliations.
