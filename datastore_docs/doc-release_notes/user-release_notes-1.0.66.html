Updates to the Ingestion Schema

Ingestion Schema Changes

We have modified the ingestion schema, in many cases using extensions and restrictions. The following section describes those changes.

Domain Entities

  • SLC-CalendarDate - Added EducationOrgReference as a key field.
  • SLC-GradingPeriod - Removed GradingPeriodIdentity added GradingPeriod, SchoolYear and EducationOrganizationReference.
  • SLC-CalendarDateIdentityType - Added an embedded EducationOrgReference.
  • SLC-SectionIdentityType - Changed UniqueSectionCode to use SLC-UniqueSectionCode.
  • SLC-UniqueSectionCode - New simple type. A unique identifier for the section, that is defined by the classroom, the subjects taught, and the instructors that are assigned.

Update to the API Schema

API Changes

No changes to the API were implemented in 1.0.66

DE585: Realm Management page - Cursor should be in first field

When opening the Manage Realm page, the cursor is not pointed to the top text box.

DE204: Should not be able to disable the admin app

The SLC Operator can disable the Admin App. That should not be allowed.

DE1589: API query strings don't properly handle a '+' as a representation of the a space character

API query strings now properly handle a '+' as a representation of the a space character.

DE1526: Incorrect 404 status being returned when an optionalFields parameter is provided to the API

The API now returns more meaningful error codes when incorrect parameters are provided.

DE2122: Application Approval can be spammed

In the Application Authorization tool, when a SEA/LEA super admin tries to authorize any application and then clicks on the button quickly multiple times, multiple application authorization entries (instead of one) are created.

DE2205: Realm creation says Realm was created successfully even if there are validation errors on the screen

Realm creation says that Realm was created successfully even if there are validation errors on the screen.

DE2019: Admin delegation shows no success message on save

When a permission is delegated to or revoked from SEA, upon clicking on Save, the user does not get a message that the change is actually saved.

DE2249: Patch method allows updates on natural keys

PATCH requests cannot update natural key fields.

DE2172: v1/schools returns object instead of array

Due to url rewriting, the request for v1/schools becomes v1/schools/{id}, which is then resolved to a single entity, instead of a List.

DE2275: Users with WRITE_GENERAL can modify WRITE_RESTRICTED fields

Users with WRITE_GENERAL can modify fields that require WRITE_RESTRICTED.

DE2006: Request of /api/rest/v1 with no resource results in 500 (IndexOutOfBoundsException)

Request of /api/rest/v1 with no resource results in 500 (IndexOutOfBoundsException).

DE2181: Deny Student Access Through Program/Cohort When StudentRecordAccess Does Not Exist

The system allows access of students to Staff/Teacher through Program or Cohort if StudentRecordAccess flag does not exist in the StaffProgramAssociation or StaffCohortAssociation in the source data. It should deny.

DE2156: Response data for API calls to api/rest/adminDelegation/myEdOrg contain full entity data, not just the body

The API call that gives an info about what has been delegated by an LEA (api/rest/adminDelegation/myEdOrg) reveals unnecessary additional information.

DE2155: SEAs can't see delegation info when only security event viewing is delegated

The API call that returns what have been delegated (from the LEA) to SEA, gives no info (empty list) in a case when only the viewing of security events has been delegated to the SEA.

DE1832: Dev without "Ingestion User" role can still ingest data

An application developer has access to the Landing Zone provisioning tool, and can provision a landing zone but can not access it via SFTP. Therefore can not ingest data manually but can do so through auto provisioning.

DE1722: API call - /api/rest/tenant returns all tenant information

In Sandbox and Production if a logged in developer using his/her token makes the /api/rest/tenant call, information for all of the tenants will be returned.

DE2162: Sandbox Administrator cannot see Admin tab in Portal

As a workaround, you can make sure that the Sandbox Administrator also explicitly has application developer role.

DE1876: Posting an entity twice results in API 500

Posting an entity twice would result in an API 500 error. We've changed features of the natural keys calculations which caused this error. We have fixed this defect.

DE921: Duplicates are not reported

Error reports are not always clear regarding the records persisted. When there are duplicates within the XML file, or a record already exists, the ingestion engine will still update the record. However, this record count is not included in the Ingested records count.

DE1458: Handle more than one teacher of record

The ingestion service can currently accept more than one teacher for a single section. The dashboard does not currently support this behavior.

DE157: School and EdOrg only supports a single representation of parentEducationAgencyReference

Reference resolution is only supported as the default scenario that features no exceptional or error conditions. As a result, a user may ingest valid EdFi format with improper references that would fail to resolve.

DE325: Global uniqueness of ProgramIdentifier is assumed when multiple EdOrgs can use the same IDs

Ingestion process assumes that the Program ID is unique within a tenant, but this should be unique within the specified EdOrg instead. As a result, the reference could resolve incorrectly.

DE273: DisciplineDescriptor not currently supported in Interchange-StudentDiscipline

This descriptor is not currently supported. However, a reference to this descriptor is required for the DisciplineAction entity.

DE198: EducationOrganization references to Program not properly ingested

This reference hasn't been implemented yet. As a result, the API won't provide a link from the EdOrg to the Program.

DE1434: Dashboard Builder - Drag and drop on release doesn't always return the page to the right state

When a large dataset is available, the initial dashboard view may only display a subset of the data. The dashboard currently does not display a message explaining that only a subset of data is displayed.

DE1647: Dashboard may not display all data

When a large dataset is available, the initial dashboard view may only display a subset of the data. The dashboard currently does not display a message explaining that only a subset of data is displayed.

DE1457: Section Profile - Handle 403 error when making teacher call

Currently, making a teacher call for a section with two "teachers of record" returns a 403 error. This error can cause the sectionInfo panel to not display.

DE1573: Remove Attendance Rate

The current attendence record includes an attendance rate column that may not be accurate for some use cases. The column will be removed by default in a future release.

DE1760: Including the word Script in data results in blacklist filter error.

If your data includes the word "script," then the user will be added to a blacklist filter.

DE162: Course name (not course code) is used to uniquely identify and refer to Course entities

Course entities are uniquely identified by course name within an education organization, rather than by course code.

DE710: Section, school and session references in AttendanceEvent are ignored

The way AttendanceEvent entities are handled ignores explicit characterization of the attendance event - by school, section, session. As a result, AttendanceEvents for students attending multiple schools will not properly persist.

DE2160: Dashboard: Select filter after listing section removes all student data

When using SLC Dashboard and viewing a list of students, in some cases choosing a filter related to that student's "special needs" status will result in a "no students found" error.

DE1871: Data migration scripts not completely migrating application data to new sandbox

A developer logs into the new sandbox in the sandbox environment realm and does not see their application. Workaround: The developer can see the application in the SLC realm. The developer can then click the edit button and re-save the application so it appears in the sandbox realm.

DE2173: Truncation limit for ingestion errors and warnings misconfigured in SLI Properties

Each error file is truncated after the maximum number of warnings and the maximum number of errors have been reached. The maximum number of errors is 10000 per input file. The maximum number of warnings is 10000 per input file.

DE2314: Admin or API should handle invalid EdOrgs when saving an application in the Sandbox

When an Application Developer saves an application in the Sandbox and that application has an existing allowed_edorg that no longer exists, then you get a 400 error "edorg is outside your tenant".

Web Browser Compatibility

SLC educator dashboard is best viewed in Internet Explore 8+, Firefox 6+, Safari 4+ with 1024x768 as the minimum recommended screen resolution.
